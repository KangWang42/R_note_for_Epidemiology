Oh My OpenCode: OpenCode 增强插件

机器学习与AI
AI 工具
Published

January 20, 2026

什么是 Oh My OpenCode

Oh My OpenCode 是 OpenCode 的编排层插件，由 YeonGyu Kim 开发。它在 OpenCode 基础上提供了更强大的代理系统、自动化钩子和增强的 MCP 集成。

核心特性

特性 说明
🤖 智能代理 Planner-Sisyphus 等专业化代理，增强规划和执行能力
🪝 20+ 内置钩子 自动化工作流，包括任务管理、上下文管理、会话恢复
🔌 MCP 增强 内置 Context7 文档获取和 grep.app 代码搜索
💻 LSP 支持 完整的语言服务器协议支持，支持代码分析和重构
🔧 实验性功能 抢先压缩、自动恢复等前沿特性

Oh My OpenCode vs 原版 OpenCode

功能 OpenCode Oh My OpenCode
基础代理 Build, Plan + Planner-Sisyphus, Librarian, Explore, Oracle
内置钩子 无 20+ 自动化钩子
MCP 集成 需手动配置 Context7, grep.app 默认启用
会话恢复 基础 自动错误恢复和会话续接
上下文管理 手动 自动监控和抢先压缩

安装指南

前置要求

  • OpenCode 1.0.133 或更高版本
  • Node.js 18+
  • npm/bun/yarn/pnpm 任一包管理器

步骤 1：安装 OpenCode

如果尚未安装 OpenCode：

curl -fsSL https://opencode.ai/install | bash

或使用包管理器：

npm install -g opencode-ai

步骤 2：安装 Oh My OpenCode

推荐使用 bunx（官方推荐）：

bunx oh-my-opencode install

或使用其他包管理器：

# npm
npm install -g oh-my-opencode

# bun
bun install -g oh-my-opencode

# yarn
yarn global add oh-my-opencode

# pnpm
pnpm add -g oh-my-opencode

步骤 3：启动 OpenCode

安装完成后，在项目目录启动 OpenCode：

cd your-project
opencode

Oh My OpenCode 会自动作为插件加载。

验证安装

# 检查 OpenCode 版本
opencode --version

# 启动后查看是否有 Oh My OpenCode 代理可用

配置文件

配置文件位置

位置 优先级 作用范围
.opencode/oh-my-opencode.json 高 项目级
~/.config/opencode/oh-my-opencode.json 低 全局

项目级配置优先于全局配置。

完整配置示例

{
  "agents": {
    "planner-sisyphus": {
      "enabled": true,
      "replace_plan": true
    },
    "librarian": {
      "enabled": true
    },
    "explore": {
      "enabled": true
    },
    "oracle": {
      "enabled": true
    }
  },
  "disabled_hooks": ["comment-checker"],
  "disabled_mcps": [],
  "lsp": {
    "typescript-language-server": {
      "command": ["typescript-language-server", "--stdio"],
      "extensions": [".ts", ".tsx"],
      "priority": 10
    }
  },
  "experimental": {
    "preemptive_compaction_threshold": 0.85,
    "auto_resume": false
  }
}

内置代理

Oh My OpenCode 包含四个专业化代理：

Planner-Sisyphus（默认启用）

默认的规划和执行代理，提供智能任务分解和系统化执行能力。

特性：

  • 智能规划：将复杂任务分解为可管理的步骤
  • 系统执行：逐步执行计划，带有错误处理
  • 任务续接：即使中断也能确保任务完成
  • 上下文感知：在整个规划和执行过程中保持上下文

配置：

{
  "agents": {
    "planner-sisyphus": {
      "enabled": true,
      "replace_plan": true
    }
  }
}

replace_plan: true 时会将原版 Plan 代理降级为子代理。设为 false 可同时保留两者。

Librarian（文档专家）

专注于文档和代码探索的代理，帮助查找相关代码、理解代码库。

适用场景：

  • 查找相关代码示例和模式
  • 理解代码库结构和组织
  • 管理和整理文档
  • 回答关于代码功能的问题
  • 发现相关代码和依赖

配置：

{
  "agents": {
    "librarian": {
      "enabled": true
    }
  }
}

与 Context7 MCP 配合使用效果最佳。

Explore（代码探索）

专为代码探索和导航设计的代理，高效浏览和理解大型代码库。

特性：

  • 代码导航：快速浏览代码库
  • 模式发现：发现代码中的模式和约定
  • 依赖分析：理解代码依赖和关系
  • 代码库概览：获取项目结构的高层视图

配置：

{
  "agents": {
    "explore": {
      "enabled": true
    }
  }
}

Oracle（问答专家）

基于代码库上下文的问答代理，帮助理解复杂概念。

特性：

  • 上下文感知回答：基于代码库回答问题
  • 代码解释：解释代码如何工作以及为什么这样设计
  • 最佳实践：基于代码分析提供建议
  • 故障排除：帮助诊断和解决问题

配置：

{
  "agents": {
    "oracle": {
      "enabled": true
    }
  }
}

钩子系统（Hooks）

Oh My OpenCode 包含 20+ 内置钩子，实现工作流自动化：

任务管理钩子

钩子 说明
todo-continuation-enforcer 确保任务完成，监控待办事项
empty-task-response-detector 检测空响应或不完整响应

上下文管理钩子

钩子 说明
context-window-monitor 监控上下文窗口使用情况
compaction-context-injector 智能压缩旧上下文
preemptive-compaction 在达到限制前抢先压缩（阈值可配置）
anthropic-context-window-limit-recovery 处理 Anthropic 特定的上下文限制

会话管理钩子

钩子 说明
session-recovery 自动从错误、中断中恢复
session-notification 会话事件通知
auto_resume 【实验性】恢复后自动续接会话

代码质量钩子

钩子 说明
comment-checker 验证代码注释
thinking-block-validator 验证思考块格式

输出管理钩子

钩子 说明
grep-output-truncator 截断大型 grep 输出
tool-output-truncator 管理工具输出大小

工作流钩子

钩子 说明
directory-agents-injector 根据目录结构注入代理
directory-readme-injector 自动添加 README 上下文
rules-injector 注入自定义规则

禁用钩子

在配置中使用 disabled_hooks 禁用特定钩子：

{
  "disabled_hooks": [
    "comment-checker",
    "agent-usage-reminder"
  ]
}
Tipstartup-toast 和 auto-update-checker

startup-toast 是 auto-update-checker 的子功能。要只禁用启动通知但保留更新检查，添加 "startup-toast" 到 disabled_hooks。要完全禁用更新检查，添加 "auto-update-checker"。

MCP 集成

Oh My OpenCode 内置两个 MCP 服务器，默认启用：

Context7（文档获取）

获取最新的官方库文档，确保始终能访问最新文档。

特性：

  • 始终获取最新官方文档
  • 支持流行库和框架
  • 根据代码自动获取相关文档
  • 无缝集成到工作流

使用场景：

  • 编码时查阅 API 文档
  • 理解库的使用模式
  • 查找正确的语法和参数
  • 发现库的最佳实践

grep.app（代码搜索）

超快速搜索数百万个公共 GitHub 仓库，即时发现代码示例和解决方案。

特性：

  • 超快搜索：秒级搜索数百万仓库
  • 模式发现：了解他人如何解决类似问题
  • 代码示例：发现真实世界的代码示例
  • 最佳实践：从生产代码中学习

使用场景：

  • 查找特定模式的代码示例
  • 了解流行库的使用方式
  • 从生产代码库学习
  • 寻找常见问题的解决方案

禁用 MCP

{
  "disabled_mcps": ["context7", "grep_app"]
}

LSP 配置

完整的语言服务器协议支持：

配置示例

{
  "lsp": {
    "typescript-language-server": {
      "command": ["typescript-language-server", "--stdio"],
      "extensions": [".ts", ".tsx"],
      "priority": 10
    },
    "pylsp": {
      "command": ["pylsp"],
      "extensions": [".py"],
      "priority": 5
    },
    "r-languageserver": {
      "command": ["R", "--slave", "-e", "languageserver::run()"],
      "extensions": [".R", ".r", ".rmd"],
      "priority": 5
    }
  }
}

LSP 选项

选项 说明
command 启动 LSP 服务器的命令和参数
extensions 该服务器处理的文件扩展名
priority 优先级（多个服务器处理同一文件时使用）
env 传递给服务器的环境变量
disabled 设为 true 禁用服务器

实验性功能

Warning实验性功能警告

这些功能是实验性的，可能导致意外行为。仅在理解影响后启用。

配置选项

{
  "experimental": {
    "preemptive_compaction_threshold": 0.85,
    "truncate_all_tool_outputs": false,
    "aggressive_truncation": false,
    "auto_resume": false,
    "dcp_for_compaction": false
  }
}
选项 默认值 说明
preemptive_compaction_threshold 0.85 触发抢先压缩的阈值（0.5-0.95）
truncate_all_tool_outputs false 截断所有工具输出（不仅是白名单工具）
aggressive_truncation false 超出 token 限制时激进截断
auto_resume false 恢复成功后自动续接会话
dcp_for_compaction false 启用 DCP 动态上下文裁剪

更新

npm update -g oh-my-opencode

bun update -g oh-my-opencode

卸载

完全卸载还需删除配置文件：
